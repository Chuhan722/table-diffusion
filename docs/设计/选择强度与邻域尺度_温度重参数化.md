# 选择强度 β 与邻域尺度 h：温度重参数化设计

> 归档日期：2026-07-24
> 状态：设计讨论中，**尚未实现**（遵守 CLAUDE.md：未经允许不实现后续阶段）
> 适用范围：抽样打分公式 `ℓ = β·F − d²/(2h²)` 中的 β（选择强度）与 h（邻域尺度）
> 依据数据：`scripts/probe_beta_scale.py` 探针实测（nltcs，16181×1001 查询，marginal init）

## 1. 背景与动机

参考记录抽样按 softmax 打分选 donor（见 `抽样分数+抽样概率.pdf`、`sampling.py`）：

```
ℓ_ik = β·F(z_k) − d(x_i, z_k)² / (2h²)
probs = softmax_k(ℓ_ik)              # 每行（每条当前记录）归一化
```

- **F(z_k)**：候选记录 k 的适应度（往哪个方向改能减小残差），来自 `fitness.py`。
- **d(x_i, z_k)**：当前记录 i 与候选 k 的归一化 Hamming 距离，∈[0, 1]，来自 `distance.py`。
- **β**：选择强度，放大适应度项——越大越贪心地只挑高适应度候选。
- **h**：邻域尺度，缩放距离惩罚——越小越偏向近邻（就近抄），越大距离越无所谓。

调参目标（见 PROJECT_STATUS.md「调参路线」）：五个参数最终都做「探索→精修」调度。
本文件专门处理 β 和 h，因为二者**不是独立旋钮，而是强耦合的一对**，需要先想清楚
它们的关系，才能决定怎么调度。

## 2. 核心洞察：β 和 h 搅在一起，其实只有两个自由度

softmax 只看 logit 的**相对差异**，不看绝对值。把打分式子重新审视，它其实只编码了
两件独立的事：

1. **相对权重**——适应度项 vs 距离项，谁在选择中说了算。由 β 与 `1/(2h²)` 的**比值**决定。
2. **温度 / 整体尺度**——把整个 logit 放大多少倍，决定 softmax 有多尖（多贪心）。

现状的问题：β 和 h 把这两件事**搅在了一起**。改 β 同时动了「相对权重」和「适应度侧的尺度」；
改 h 同时动了「相对权重」和「距离侧的尺度」。所以单独调任何一个都会牵连另一件事，
调参空间是拧巴的。

## 3. 探针实测：两项的真实量级（`probe_beta_scale.py`）

为把「相对权重」量化，探针实测了适应度 F 的动态范围，与距离项对比。

### 3.1 距离项的范围（解析可得）

d ∈ [0, 1]（归一化 Hamming），所以 `d²/(2h²)` 的最大值 = `1/(2h²)`：

| h   | 距离项跨度 `1/(2h²)` |
|-----|----------------------|
| 0.2 | 12.5   |
| 0.4 | 3.13   |
| 0.8 | 0.781  |（当前 baseline）
| 1.6 | 0.195  |
| 3.2 | 0.049  |

**距离项跨度随 h 平方反比变化。** h 变 16 倍（0.2→3.2），距离项跨度变 256 倍。

### 3.2 适应度 F 的范围（实测，marginal init，baseline 参数推进）

在真实运行中每隔若干轮打印 F 分布（跨度 = max−min）：

| 轮   | loss    | F 跨度 | F std | 平衡 β(=距离项跨度/F跨度, h=0.8) |
|------|---------|--------|-------|----------------------------------|
| 0    | 6.94e8  | 17.15  | 2.24  | 0.046 |
| 50   | 4.56e8  | 16.45  | 2.17  | 0.048 |
| 100  | 3.60e8  | 14.34  | 2.23  | 0.054 |
| 200  | 2.64e8  | 12.58  | 2.02  | 0.062 |
| 350  | 2.14e8  | 11.13  | 1.87  | 0.070 |
| 500  | 1.95e8  | 10.34  | 1.76  | 0.076 |

**关键结论：**

1. **F 跨度（~10–17）比距离项（h=0.8 时 0.78）大约 15–20 倍。** 说明在 baseline β=1
   下，`β·F` 完全压倒距离项——选择几乎只看适应度，**近邻约束基本被淹没**。当前
   baseline 很可能落在「适应度主导饱和区」，原计划的线性网格 [0.5,1,2,4] 会全挤在
   这个饱和区里，扫出平线、浪费机时。
2. **要让两项势均力敌，需要 β ≈ 0.05 量级**（h=0.8 时），比 baseline 小一个数量级。
   有信息的响应区间在小 β 侧，网格必须跨数量级（对数尺度）。
3. **F 会随收敛温和塌缩**：500 轮里跨度 17→10（降约 40%），但 std 几乎不动
   （2.24→1.76）——跨度降主要来自极值收敛，分布主体稳定。平衡 β 相应从 0.046 爬到
   0.076（约 1.6×）。**无剧烈塌缩**，不会出现「后期距离项完全接管」的失控。

### 3.2b F 的量级随查询数怎么长（`probe_f_scaling.py`，修正一个错误直觉）

F = Σ_j 各查询贡献。曾有一个过于干净的直觉：每查询贡献均值为 0、有正有负半抵消，
故按**独立随机游走** F ∝ √m。探针实测**推翻了它**——在同一张初始表上抽不同大小的
查询子集量 F：

| 查询数 m | F std | std/√m |
|---------|-------|--------|
| 50    | 0.221 | 0.031 |
| 100   | 0.365 | 0.037 |
| 250   | 0.656 | 0.042 |
| 500   | 1.254 | 0.056 |
| 1001  | 2.243 | 0.071 |

若严格 ∝√m，`std/√m` 应为常数；实测它从 0.031 爬到 0.071（翻倍）。m 涨 20 倍（50→1001）
时 std 涨约 10 倍，而 √m 只预测 4.5 倍——**F 增长介于 √m 和 m 之间（约 m^0.75）**。

**原因：查询之间不独立、有正相关。** 许多查询问相关的事（如「age≥65」与「age≥65 且
有医保」高度重叠），一条记录在 A 上加分往往在相关的 B 上也加分。正相关量累加比独立
情况快、趋向线性。相关性越强，越偏线性。

**关键推论：F 的量级取决于查询的「数量」和「相关结构」两者，无法用公式外推，只能实测。**
换数据集、换查询集，倍数都变。所以 γ 必须按每个数据集+查询集实测校准——这个结论
比 §3.2 更强：不是「建议实测」，而是「除了实测别无办法」。

### 3.3 β 与 h 的定量耦合关系

由 3.1/3.2 得「平衡 β」（让 β·F 跨度 ≈ 距离项跨度）：

```
β_平衡 ≈ (1/(2h²)) / F跨度 ≈ (1/(2h²)) / 17 ≈ 0.029 / h²
```

这条式子把「β 和 h 强耦合」讲死了：**它们联合起来只确定了「相对权重」这一个量。**
单独扫 β（锚定某个 h）得到的最优 β，换个 h 就失效——这正是 OAT（一次一参）扫描的
交互盲区的活例子。

## 4. 设计方案：温度重参数化

既然真正起作用的是「相对权重」和「温度」两个自由度，就直接用这两个量参数化，
把 β、h 消掉：

```
ℓ_ik = λ · ( F(z_k) − γ · d(x_i, z_k)² )
```

- **γ（gamma）= 相对权重**：距离相对适应度的重要性。**一个标量，先固定。**
- **λ（lambda）= 逆温度**：唯一的调度旋钮。
  - λ 小 → logit 平 → softmax 近均匀 → 什么都可能抽 → **探索**
  - λ 大 → logit 尖 → 只抽联合得分最高的 → **精修**

### 4.1 与原参数的换算

对照 `ℓ = β·F − d²/(2h²)` 与 `ℓ = λ(F − γd²) = λF − λγd²`：

```
β = λ
1/(2h²) = λγ   ⟹   h = 1 / √(2λγ)
```

看 λ 从小到大（调度推进）时（固定 γ）：

| λ      | β    | h    | 效果          |
|--------|------|------|---------------|
| 小(前期) | 小   | 大   | 选择弱、邻域宽 → 广探索 |
| 大(后期) | 大   | 小   | 选择强、邻域紧 → 精收敛 |

**β 升、h 降，两者一起朝「探索→精修」走，全由 λ 一个旋钮驱动。**

### 4.2 与既有设计的印证

`选择强度+邻域尺度.pdf` / 原方案里「h 前期 0.8 → 后期 0.15（线性衰减）」的直觉，
在本方案里被自动满足——温度退火（λ 增）天然导出 h 递减。原设计方向是对的，
本方案只是找到了统一 β、h 的方式，并给出了 β 也该同向递增这一新结论。

## 5. 争议点：固定 γ 是否合理？

这是本设计的核心假设，也是最需要论证的地方。**结论：固定 γ 是站得住的起点，
但不完美——存在温和的、方向正确的漂移，先固定、后期再按需补偿。**

### 5.1 反对固定 γ 的理由

1. **探索/精修阶段对「就近」的需求本就不同。** 前期表粗糙，大改结构是好事，不该
   太在乎就近（γ 宜小）；后期结构成形，大跳破坏已成结构（γ 宜大）。即 γ 天然想随
   进度 p 递增，和 λ 同向。
2. **隐式漂移。** F 随收敛塌缩（§3.2），而距离项恒在 [0,0.78]。F 一缩，固定 γ 下
   距离项的相对话语权就自动上升——「你以为固定了，其实没固定」。

### 5.2 数据给的答复

探针跑到 500 轮（§3.2）实测了这个漂移：

- **无剧烈塌缩**：F 跨度到 500 轮还有 10，稳稳压着距离项 0.78，不会失控。
- **漂移温和且方向正确**：平衡 β 从 0.046→0.076（1.6×），方向是「距离项话语权随
   收敛缓慢上升」——恰与 §5.1 第 1 点的第一性直觉同向。也就是说，固定 γ 只是
   「稍微保守」，不会把方向搞反。
- **量级上，1.6× 相比 β 本身要扫的两个数量级（0→1）是小量**，固定 γ 的近似误差可接受。

### 5.3 三个层次的方案（复杂度递增）

| 方案 | 自由度 | 说明 |
|------|--------|------|
| **A. 固定 γ + 调度 λ** | 1 曲线 + 1 常数 | 最简。忽略 γ 的温和漂移，但数据表明误差小、方向不错 |
| B. γ 也随 p 独立调度 | 2 曲线 | 最灵活，但回到「两个旋钮」，失去简洁 |
| C. γ = λ 的函数（耦合） | 1 曲线 | γ 跟着温度走（随 λ 单调升），仍是一个驱动量，补偿漂移 |

**采纳方案 A 作为起点**：先用固定 γ 把温度退火跑通、验证「退火 vs 固定参数」是否
真有收益；若后期发现 γ 的漂移影响明显，再升级到方案 C。这符合「先简单，再复杂」。

### 5.4 方法论提醒

§2–§5 大量是纸上推理。要警惕过度设计——为尚未被数据证实的问题堆机制。§3.2 把探针
延长到 500 轮，正是用最小实验切断「固定 γ 会不会失控」这条推理链的做法。后续每加一层
复杂度（A→C），都应先有数据支撑，而非仅靠推演。

## 6. γ 怎么设计：不是硬编码常数，而是「实测尺度 × 无量纲旋钮」

§3.2b 得出：F 的量级只能实测、换数据就变。这直接决定了 γ 的设计——**γ 不能写死成
一个数（如 13），否则换数据集就失效。** 正确做法是把 γ 拆成两部分：

```
γ = k · S_F        （其中 S_F = 实测的 F 尺度，k = 无量纲平衡旋钮）
```

- **S_F（F 尺度）= 探针实测量**：距离项 d² 跨度固定为 1（d∈[0,1]），F 跨度约 10–17。
  取「全程 F 跨度的中位」作 S_F（本数据集 ≈ 13）。**换数据集就重跑探针，S_F 自动更新。**
- **k（平衡旋钮）= 无量纲、跨数据集可移植的真正待调量**：
  - k = 1 → 距离项跨度 ≈ F 跨度，两项势均力敌（平衡点）
  - k < 1 → 距离项弱，适应度主导（偏探索方向的选择）
  - k > 1 → 距离项强，就近主导（偏保守）

**这样设计的好处：**

1. **可移植**。要扫、要调的是 k（无量纲比例，语义是「相对权重」），不是 γ（带数据尺度
   的绝对值）。k 的甜点大概率跨数据集稳定（都在 1 附近），γ 则每个数据集不同——由 S_F
   吸收数据差异。
2. **校准与调参解耦**。S_F 由探针一次性测定（`probe_beta_scale.py` 已能给），k 由扫描找。
   换数据只需重测 S_F，不必重扫 k。
3. **对上 §5.2 的漂移**。F 随收敛从 17 塌到 10。若 S_F 取固定中位 13，则前期距离项被
   稍压、后期被稍放——就是那 1.6× 的温和漂移，方案 A 接受它。若将来要消除，可让 S_F
   跟着实时 F 尺度走（等价方案 C 的一种实现）。

**所以「扫 γ」这一步的正确形式是「扫 k」**：固定 S_F=13，扫 k ∈ {0（纯适应度对照）,
0.25, 0.5, 1, 2, 4} 之类，看 loss 对相对权重的响应，找甜点。k 无量纲，网格跨数据集通用。

注：这里的 γ 是「F − γ·d²」形式下 d² 的系数。与 §3.3 的 `β_平衡≈0.029/h²` 是同一件事的
不同归一方式（那里尺度归到 β，这里归到 γ 并显式拆出实测的 S_F）。

## 7. 对调参计划的影响

原计划：逐个 OAT 扫 β、h（各一维），再对 β×h 做二维网格（担心交互）。

**本方案把 β、h 合并**，调参塌成：

0. **实测 S_F**（探针，一次性）——`probe_beta_scale.py` 已给：本数据集 F 尺度中位 ≈ 13。
   换数据集重跑这一步。
1. **扫无量纲旋钮 k**（一维，替代原来的「扫 β」这一步）——固定 γ=k·S_F，扫 k 找相对
   权重甜点。k 无量纲、跨数据集可移植，网格如 {0, 0.25, 0.5, 1, 2, 4}，其中 k=0 是
   「纯适应度、关掉距离」的对照。
2. **设计 λ(p) 退火曲线**——从热到冷，由进度比例 p=t/n_rounds 驱动（见 PROJECT_STATUS.md
   「调度驱动量」）。这就是 β、h 的「探索→精修」调度。

五个待调度参数里，β 和 h 合并为「γ=k·S_F（固定）+ λ（调度）」，少一个调度自由度，
语义也更干净，且 k 跨数据集通用。
